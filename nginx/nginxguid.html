Nginx 웹 서버 역할

1. 클라이언트 요청 수신

웹 브라우저(클라이언트)에서 웹 사이트 URL을 입력하면, Nginx는 해당 요청을 수신합니다.

2. 정적 컨텐츠 제공

요청된 URL이 정적 파일(HTML, CSS, JavaScript, 이미지 등)인 경우, Nginx는 직접 해당 파일을 클라이언트에게 제공합니다.

3. 동적 컨텐츠 전달

요청된 URL이 동적 컨텐츠(PHP, JSP, ASP 등)인 경우, Nginx는 해당 요청을 웹 애플리케이션 서버(WAS)로 전달합니다.

4. WAS에서 처리된 응답 수신

WAS에서 동적 컨텐츠를 처리한 후, Nginx는 처리 결과를 클라이언트에게 전달합니다.
