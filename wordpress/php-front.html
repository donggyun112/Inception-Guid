Docker 컨테이너에서 PHP-FPM 실행

Docker 컨테이너에서 PHP-FPM을 실행할 때는 포그라운드 모드로 실행해야 합니다. 그렇지 않으면 PHP-FPM 프로세스가 종료되면 컨테이너도 종료됩니다.

Docker 컨테이너는 실행 중인 포그라운드 프로세스가 있어야 합니다. 만약 포그라운드 프로세스가 종료되면, Docker는 컨테이너가 더 이상 실행 중이지 않다고 판단하고 컨테이너를 자동으로 종료시킵니다.

PHP-FPM 포그라운드 실행 명령어

PHP-FPM을 포그라운드 모드로 실행하기 위해서는 PHP-FPM<version> -F 명령어를 사용해야 합니다. 여기서 <version>은 사용 중인 PHP-FPM의 버전을 나타냅니다.

예를 들어, PHP-FPM 7.4 버전을 포그라운드 모드로 실행하려면 다음 명령어를 사용합니다:

PHP-FPM7.4 -F

-F 옵션은 PHP-FPM이 포그라운드 모드로 실행되도록 지시합니다. 이 옵션을 사용하지 않으면 PHP-FPM은 기본적으로 백그라운드 프로세스로 실행되어 컨테이너가 종료될 수 있습니다.

Docker 컨테이너에서 PHP-FPM 실행 예시

Docker 컨테이너에서 PHP-FPM을 포그라운드 모드로 실행하는 예시 Dockerfile의 일부는 다음과 같습니다:

FROM php:7.4-fpm

# ... (생략) ...

CMD ["PHP-FPM7.4", "-F"]

위의 예시에서는 PHP 7.4 버전의 FPM 이미지를 기반으로 하고, Dockerfile의 CMD 명령어에서 PHP-FPM7.4 -F를 사용하여 PHP-FPM을 포그라운드 모드로 실행합니다.
