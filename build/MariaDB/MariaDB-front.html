MariaDB 실행 방법

1. service mariadb start 명령어 사용

service mariadb start 명령어를 사용하여 MariaDB를 실행할 수 있습니다. 이 경우 다음과 같은 장점이 있습니다:

  • /run/mysqld 폴더가 자동으로 생성됩니다.
  • 폴더에 대한 권한 설정도 자동으로 처리됩니다.

단, service mariadb start 명령어를 사용하면 MariaDB가 백그라운드에서 실행됩니다.

2. mysqld_safe 명령어 사용

mysqld_safe 명령어를 사용하여 MariaDB를 포그라운드에서 실행할 수 있습니다. 이 명령어는 다음과 같은 작업을 수행합니다:

  • MariaDB 서버 프로세스를 시작합니다.
  • 서버 프로세스가 예기치 않게 종료되면 자동으로 재시작합니다.
  • 에러 로그 파일에 로그를 기록합니다.
  • 시스템 로그에 이벤트를 기록합니다.
  • 필요한 소켓 파일을 자동으로 생성합니다.

mysqld_safe 명령어를 사용할 때는 다음과 같은 작업이 필요합니다:

  • /run/mysqld 디렉토리를 생성해야 합니다.
  • 디렉토리에 대한 권한을 mysql 사용자에게 부여해야 합니다.

예를 들어, 다음과 같이 디렉토리를 생성하고 권한을 부여할 수 있습니다: 

 mkdir /run/mysqld
 chown mysql:mysql /run/mysqld

3. 주의 사항

만약 /run/mysqld 폴더를 직접 생성하고 권한을 부여한 상태에서 service mariadb start 명령어를 실행하면, 이미 폴더가 존재하기 때문에 MariaDB가 실행되지 않습니다.

4. 정리

  • service mariadb start: 자동으로 폴더 생성 및 권한 설정, 백그라운드 실행
  • mysqld_safe: 포그라운드 실행, 수동으로 폴더 생성 및 권한 설정 필요

5. 추천방식

  • Servie mariadb start 명령어를 사용하여 MariaDB를 실행하하고 초기설정을 실행합니다.
  • 이후 Service mariadb stop 명령어를 사용하여 MariaDB를 종료합니다.
  • 마지막으로 mysqld_safe 명령어를 사용하여 MariaDB를 포그라운드로 실행합니다

    • 이 방식을 선택하게 되면 Servie mariadb start가 필요한 폴더 및 권한을 구성하기 때문에 mysqld_safe 명령어를 사용할 때 불편함이 없습니다.
